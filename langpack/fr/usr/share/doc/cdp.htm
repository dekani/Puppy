NOM

cdplay -- un programme interactif de mode de texte pour commander et jouer des ROM de CD audio sous Linux.  

SYNOPSIS

cdp [ -h | -l | -n | -s ] [play <track-num>]
cdplay [ -h | -l | -n | -s ] [play <num>] [stop] [table]

           -s : Démarrer en mode silence

           -h : aide

           -n : pas d'autoplay

           -l : démarrage lent (en attente d' init)

   play <num> : lecture automatique depuis <le n°> de piste du CD

         stop : arrêt du CD.

        table : affiche le contenu du menu du CD depuis la base de données

Saisissez cdp pour lancer le programme. Verrouillez le pavé numérique (NUM LOCK). cdp ouvert, presser la touche DEL et un menu d'aide s'affichera. Tout en tournant, cdp montrera le numéro de piste courant et la durée totale écoulée, de la piste jouée ainsi que celle du CD. Notez que le temps total écoulé du CD, inclut les durées pour les pistes, données audio et non-audio comprises.

Utilisez cdplay pour employer la version non-interactive de cdp.

 

DESCRIPTION

cdpest un programme interactif en ligne de commande pour pour écouter des CDs audio sous Linux. Il est basé sur WorkBone, lui-même dépendant des modules"hardware.c", "database.c" et "struct.c" du programme complexe de CD X11, connu sous le nom de WOrkMan. cdp montre les bases de données de CD, comme celles générées par WorkMan. Il permet également d'éditer cette information.

cdps'attend à trouver le périphérique en : /dev/cdrom. Si vous n'avez pas cette configuration, allez à /dev/nomdupériphérique et créez un lien symbolique vers /dev/cdrom. Par exemple, si vous appelez actuellement votre lecteur cdrom depuis 'mcd0', faites la chose suivante :


               cd /dev

               ln -sf mcd0 cdrom

Autrement, éditez juste 'hardware.c' pour qu'il reflète votre configuration et recompilez

cdpUtilise le clavier numérique comme un panneau de configuration pour vous permettre de vous déplacer de piste en piste, jouer, mettre en pause, arrêter et résumer les actions sur le CD audio. Assurez-vous d'avoir verrouiller votre clavier, pour l'utilisation de cdp. Un modèle graphique est affiché pour vous guider. Les temps de fonctionnement sont sans arrêt mis à jour et également affichés.



                 +---- number pad -----+

                 |                     |

                 |    7     8     9    |

                 |                     |  

                 |    4     5     6    |

                 |                     |

                 |    1     2     3    |

                 |                     |

                 |    0          del   |

                 |                     |

                 +---------------------+


la touche '9' ---> "lire"
la touche '8' ---> "pause/reprise"
la touche '7' ---> "arrêt"
la touche '6' ---> "suivant"
la touche '5' ---> "re-lire"
la touche '4' ---> "précédent"
la touche '3' ---> "avance de 15 secondes"
la touche '2' ---> "arrêt forcé" (la musique s'arrête)
la touche '1' ---> "en arrière de 15 secondes"
la touche '0' ---> "quitter le programme" (la musique continue)
la touche '.' ---> "aide"
la touche 'enter/entrée' ---> édite la chanson courante.
la touche 'a' ---> édite le nom de l'artiste.
la touche 'c' ---> édite le nom du CD.

BUGS

Il est douteux que je fasses beaucoup de développement pour cdp.J'ai retravaillé WorkBone 0.1 pour le support des bases de données de CD. Ce programme est une autre prolongation de WorkBoneII 0.2 pour soutenir l'édition de base de données de CD. Svp envoyez un E-mail pour tous fixages de bugs ou amélioriations, que vous pourriez avoir fait. cdpfonctionne avec mon lecteur de CDRom Mitsumi FX001D, mais à l'usage avec d'autres lecteurs ceci peut être différent pour vous. Seulement les appels et les fonctions génériques sont utilisés de façon appropriée, par WorkMan lui-même. À la différence de WorkBone, cdp utilise ncurses pour afficher l'écran. Si quiconque est intéressé pour un portage sur d'autres plate-formes veuillez m'envoyer un email.
