MODEM-STATS

Section : Commandes utilisateur (1)
Mise à jour : 20 August 1995
Index Retour au sommaire
 

NOM

modem-stats - Envoi d'une commande au modem et copie de la réponse en sortie-standard.  

SYNOPSIS

modem-stats [ -c command ] [ -e end_string ] device  

DESCRIPTION

modem-stats est typiquement utilisé pour documenter le statut d'un modem après une connexion et peut aussi être utilisé pour recueillir la statistique comptable.
modem-stats peut être utilisé pour envoyer une chaîne d'initialisation à un modem.
modem-stats peut aussi être utilisé pour 'reseter' un modem.  

OPTIONS

-c command
Fixer la commande à envoyer au modem.
Par défaut AT I6
Même si la command est répercutée par le modem, elle ne sera pas copiée en sortie.
-e end_string
Fixer la chaîne de réponse finale du modem.
Par défaut OK
Toutes les réponses du modem à la command,jusqu'à end_string non-inclus, seront copiées en sortie.
 

NOTES

modem-stats a besoin d'un accès lecture/écriture à l'appareil.
modem-stats quitte si l'appareil ne réponds pas dans les 5 secondes.  

EXEMPLES

 

Enregistrer le status du modem

date >> status
modem-stats /dev/cua1 >> status  

Initialiser le modem

modem-stats -c "AT &F1" /dev/cua1 > /dev/null  

Reset du modem

modem-stats -c ATZ /dev/cua1 > /dev/null  

VOIR AUSSI

Le manuel de votre modem.  

BUGS

modem-stats quitte improprement si end_string n'est pas correcte.
Tous les modems ne fournissent pas de liens statistiques utiles.
modem-stats ne fonctionne pas avec mgetty(1).
