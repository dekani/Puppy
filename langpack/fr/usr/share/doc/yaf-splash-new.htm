yaf-splash (nouveau)

L'ancien yaf-splash est disponible en paquet PET et très probablement déjà dans votre distribution. Il est situé dans /usr/X11R7/bin/yaf-splash. La documentation de l'ancien yaf-splash est dans /usr/share/doc/yaf-splash.htm.

Il y a maintenant une amélioration avec le remplaçant de yaf-splash. Il s'agit du script /usr/share/doc/yaf-splash.htm, avec un lien symbolique vers /usr/bin/yaf-splash.

Si vous exécutez 'yaf-splash', le nouveau script sera lancé. Mais, le script gtkdialog-splash lancera l'ancien (si installé) s'il détecte le paramètre "-font" dans la ligne de commande.

Le nouveau yaf-splash n'utilise pas le paramètre '-font' (il l'ignore). Voici les options de ligne de commande du nouveau yaf-splash:

 -text EXPRESSION
 -timeout COUNT (en secondes)
 -icon GTK-XXX (par exemple: gtk-info all gtk-stock-symbols, défaut: none)
 -bg COLOR (background color red, blue, yellow..., default: grey)
 -fg COLOR (coleur de police, défaut: black)
 -placement center|mouse|top|bottom|top-left|bottom-left (défaut center)
 -close never|mouseover|box (défaut is mouseover)
 -deco TITLE (affiche les décorations des fenêtres, avec titre)
 -fontsize SIZE (small, medium, large, x-large, défaut: large)

Exemple:

# yaf-splash -placement top -close box -text "This is a message"

Le nouveau yaf-splash a été initialement écrit par 'Mave', modifié par Barry Kauler.

Notez que le nouveau yaf-splash fonctionne mieux avec le gestionnaire de fenêtres JWM, qui est configuré pour traiter la classe de 'gtkdialog-splash' d'une manière particulière (voir /root/.jwmrc).
Ceux qui préfèrent l'ancien yaf-splash peuvent continuer à l'utiliser, simplement en mettant en paramètre '-font' - ce qui est toujours fait de toutes façons.
