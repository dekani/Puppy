NOM

Axel - Un accélérateur de téléchargements, léger pour Linux.

 

SYNOPSIS

axel [OPTIONS] url1 [url2] [url...]

 

DESCRIPTION

Axel est un programme qui télécharge un fichier depuis un serveur ftp/HTTP grâce à une connexion multiple, chaque connexion téléchargeant sa propre partie du fichier.

À la différence de la plupart des autres programmes, Axel télécharge toutes les données directement vers le fichier de destination, en utilisant un lien simple. Il fait gagner du temps justement à la fin parce que le programme n'a pas à enchaîner toutes les parties téléchargées.

 

OPTIONS

Un argument est exigé, l'URL du fichier que vous voulez télécharger. En téléchargeant du FTP, le nom de fichier peut contenir des caractères de remplacement et le programme essayera de résoudre le nom de fichier complet. Des URL multiples peuvent être aussi bien être spécifiées et le programme utilisera toutes ces URL pour le téléchargement. Veuillez noter que le programme ne vérifie pas si les dossiers sont similaires.

Autres options:

--max-speed=x, -s x
Vous pouvez spécifier ici, une vitesse (octets par seconde) et Axel essayera de garder la vitesse moyenne autour de cette vitesse. Utile si vous ne voulez pas que le programme consomme toute votre bande-passante.

--num-connections=x, -n x
Ici, vous pouvez spécifier le nombre de connexions différentes.

--output=x, -o x
Les données téléchargées seront mises dans un fichier local avec le même nom, à moins que vous ne spécifiiez un nom différent en utilisant cette option. Vous pouvez indiquer un répertoire aussi, le programme ajoutera le nom de fichier.

--search[=x], -S [x]
Axel peut faire une recherche de miroirs en utilisant le moteur de recherche filesearching.com. Cette recherche sera faite si vous utilisez cette option. Vous pouvez aussi spécifier combien de miroirs différents devraient être utilisés pour le téléchargement.

La recherche de miroirs peut prendre du temps car le programme évalue la vitesse de chaque serveur et il vérifie si le fichier est toujours disponible.

--no-proxy, -N
Ne pas utiliser de serveur proxy pour télécharger le fichier. Impossible bien sûr quand un proxy transparent est actif quelque part.

--verbose
Si vous voulez voir plus de messages d'informations, vous pouvez utiliser cette option. Utilisez-la plus d'une fois si vous voulez en voir plus.

--quiet, -q
Pas de sortie d'informations à l'écran.

--alternate, -a
Cela montrera un autre indicateur de progression. Une barre affiche l'avancement et le statut des différents liens , avec la vitesse courante et une évaluation du temps de téléchargement restant.

--help, -h
Un bref résumé de toutes les options.

--version, -V
Obtenir l'information sur la version.

 

NOTE

Les options longues (doubles tirets) sont gérées seulement si votre plate-forme supporte la fonction 'getopt_long'. Sinon (comme *BSD), seules les options courtes peuvent être utilisées.

 

VALEUR DE RETOUR

Le programme renvoie : 0 quand le téléchargement est réussi, 1 quand un gros problème arrive, 2 quand le téléchargement est interrompu, Si quelque chose d'autre est retournée, ça doit être un bug ..

 

EXEMPLES

axel ftp://ftp.{be,nl,uk,de}.kernel.org/pub/linux/kernel/ v2.4/linux-2.4.17.tar.bz2

Ceci utilisera les mirois : belges, hollandais, anglais et allemands de 'kernel.org' pour télécharger une image du noyau Linux 2.4.17. 

axel -S 4 ftp://ftp.kernel.org/pub/linux/kernel/v2.4/ linux-2.4.17.tar.bz2

Ceci effectuera une recherche du fichier 'linux-2.4.17.tar.bz2' sur 'filesearching.com' et utilisera les quatre (si possible) miroirs les plus rapides pour le téléchargement. (probablement comprenant ftp.kernel.org)

(naturellement, les commandes sont à saisir sur une seule ligne, ce que ne permet pas la mise en forme de cette page)

 

FILES

/etc/axelrc Fichier de configuration générale

~/.axelrc Fichier de configuration personnelle

Ces fichiers ne sont pas documentés dans la page du manuel, mais le fichier d'exemple qui est fournit avec le programme contient, j'espère, assez d'informations. L'emplacement du fichier de configuration générale, pourrait être différent.

BUGS

Je suis sûr qu'il y a encore quelques bugs, dites-le-moi s'il vous plaît et j'essayerai de les fixer.

Un bug connu consiste en ce que le programme fait des choses mystérieuses en téléchargeant par l'intermédiaire de centaines de connexions. Et bien tout simplement, ne le paramétrez pas ainsi.
